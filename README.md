# Odds Comparison Engine

## Overview

The Odds Comparison Engine is the backend component of the Odds Comparison Platform.

It continuously collects odds from supported sportsbooks, retrieves market prices from Betfair Exchange, performs event matching, calculates price differences, and writes the results into the MySQL database.

The PHP frontend only displays data generated by this engine.

The engine is the authoritative source for all comparison logic.

---

# Production Status

This repository represents a mature production system that has evolved over many years.

The architecture is considered stable.

Large refactors should be avoided unless explicitly requested.

Backwards compatibility is preferred whenever practical.

---

---

# Responsibilities

The engine is responsible for:

- Betfair API communication
- Sportsbook scraping/API collection
- Event matching
- Fuzzy team matching
- Odds comparison
- Arbitrage calculations
- Writing comparison data
- Scheduling
- Background processing

The engine is **not** responsible for presentation or user interface logic.

# Main Components

## gamma/

Contains the runtime engine.

Important files include:

- arb_monitor.py
- history_preprocessor.py
- mysql_interface.py

---

## arber_modules/

Contains one module per provider.

Examples:

- betfair.py
- unibet.py
- bet3000.py

Each provider is intentionally isolated.

---

## pre_vwap/

Contains historical VWAP implementations.

These are preserved for compatibility.

---

## certs/

Contains Betfair certificates.

---

# Main Orchestrator

gamma/arb_monitor.py controls:

- Betfair login
- Provider execution
- Competition loading
- Event processing
- Thread scheduling
- Database updates

This file should be modified carefully.

---

# Database

The engine owns database writes.

The frontend primarily performs reads.

Business rules should remain inside the engine.

---

# Design Principles

- Stability over cleverness.
- Small changes over rewrites.
- Preserve production behaviour.
- Prefer extending existing modules.
- Avoid duplicate logic.
