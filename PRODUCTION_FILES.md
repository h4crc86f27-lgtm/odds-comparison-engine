# PRODUCTION FILES

Version: 1.0

Status: Production

This document identifies which files are part of the active production
engine and which files exist only for historical, testing or backup
purposes.

Future developers and AI assistants should use this document before
making changes.

------------------------------------------------------------
1. PRODUCTION SOURCE CODE
------------------------------------------------------------

The following files are considered production code.

Main Engine

gamma/arb_monitor.py

Historical Processing

gamma/history_preprocessor.py

Provider Modules

gamma/arber_modules/

Configuration

gamma/config.ini

Documentation

README.md
AGENTS.md
ENGINE_ARCHITECTURE.md
ENGINE_OVERVIEW.md
CODEMAP.md
FILE_INDEX.md

------------------------------------------------------------
2. ACTIVE PROVIDERS
------------------------------------------------------------

The production engine currently supports:

• Betfair
• Unibet
• Bet3000

These are the only providers currently maintained.

------------------------------------------------------------
3. LEGACY PROVIDERS
------------------------------------------------------------

Legacy provider modules remain in the repository for compatibility and
historical reference.

Examples include:

• Bingoal
• Contra
• QRBet
• TOTO
• Winkel Toto
• Yess365
• Bet635
• BetAlpha
• Live90Bet
• M8Bets

Most are currently inactive because external APIs are no longer
available.

Do not remove these modules without explicit approval.

------------------------------------------------------------

------------------------------------------------------------

------------------------------------------------------------
6. LOG FILES
------------------------------------------------------------

Examples:

*.log
nohup.out

These files contain runtime logs.

Do not modify them.

------------------------------------------------------------
7. GENERATED FILES
------------------------------------------------------------

Examples:

*.pickle
*.json

These are generated by the running engine.

They should normally not be edited manually.

------------------------------------------------------------
8. DATABASE EXPORTS
------------------------------------------------------------

Examples:

*.sql
*.zip

These are database backups.

They are not source code.

------------------------------------------------------------
9. MEDIA FILES
------------------------------------------------------------

Examples:

*.png

These are screenshots or diagnostic assets.

------------------------------------------------------------
10. AI DEVELOPMENT RULES
------------------------------------------------------------

When modifying this repository:

• Only modify active production files.

• Ignore backup files.

• Ignore historical snapshots.

• Ignore generated pickle files.

• Ignore SQL backups.

• Ignore runtime logs.

• Ignore screenshots.

If multiple versions of the same file exist, always use the active
production file unless explicitly instructed otherwise.

------------------------------------------------------------
11. PRODUCTION FIRST
------------------------------------------------------------

The active production engine is intentionally conservative.

Future improvements should focus on:

• Stability

• Reliability

• Documentation

• Maintainability

Large architectural redesigns should not be proposed unless explicitly
requested.

End of document.
